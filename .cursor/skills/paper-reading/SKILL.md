# Paper Reading — 论文深度阅读与分析

## 输入获取

1. **用户提供 PDF 文件路径或 PDF 链接时**：先使用 `pdf-to-markdown` skill 将 PDF 转为 Markdown，再读取 Markdown 内容进行分析。具体步骤：
   - 如果是 URL，先下载到 `temp/` 目录（arXiv 链接需将 `/abs/` 替换为 `/pdf/`）
   - 运行转换脚本：`uv run python .cursor/skills/pdf-to-markdown/scripts/pdf2md.py <pdf_path> -o temp/<name>.md`
   - 用 `Read` 工具读取生成的 Markdown 文件
2. 用户提供论文 URL（HTML 页面，非 PDF）时，用 `WebFetch` 获取内容
3. 用户直接粘贴文本时，直接使用

如果论文内容过长无法一次获取完整，分段读取并合并。

## 输出规范

### 文件命名与路径

- 默认文件名：`<Paper_Title_用下划线连接>.md`
- 输出路径：**询问用户**希望写入的目录，若用户不指定则写入当前工作区根目录
- 若同名文件已存在，询问是否覆盖

### 数学公式格式

**严格遵守如下的的规则**：

- 行内公式：`$ formula $`，两侧与文字之间保留空格
- 块级公式：`$$` 独占一行，前后各空一行

一个典型的例子如下:
```markdown
其中 $ p_i $ 是注意力权重，维度为 $ d $。

$$
L = -\sum_{t=1}^{T} \log p(x_t \mid x_{<t})
$$
```

## 写作要求

### 翻译

- 非中文论文必须**全文翻译为中文**
- 保留原文章节编号和结构层次
- 专业术语首次出现时给出英文原文，如：向量量化（Vector Quantization）
- 人名、模型名、数据集名保留英文原文

### 数学公式展开

这是最重要的部分。对论文中每个非平凡公式：

1. **符号逐项解释**：列出公式中每个符号的含义和维度
2. **推导过程**：从前提假设出发，展示推导的关键步骤；论文省略的中间步骤要补全
3. **直觉理解**：用类比、几何解释或极端情况分析帮助理解
4. **与已知概念关联**：与读者可能熟悉的概念建立联系（如：与 GPT 的类比、与贝叶斯定理的关系）

使用引用块 `>` 包裹数学原理详解，与翻译正文在视觉上区分。

### 核心与重点部分

识别并重点展开以下内容（不限于此）：

- 论文提出的核心创新点和新方法
- 关键损失函数的设计动机和数学推导
- 模型架构的设计选择及其原因
- 训练技巧和工程细节
- 实验中的关键发现和消融实验

### 方案演进对比

- 使用表格对比，至少覆盖 3-5 个相关工作
- 时间跨度涵盖**之前、同时期、之后**的工作
- 对比维度包括但不限于：核心思路、技术路线、实验规模、优势、局限
- 末尾给出演进趋势总结

- [ ] 全文已翻译为中文（术语首现附英文）

- [ ] 方案演进对比表已填写完整
- [ ] 章节编号与原文一致
- [ ] 参考文献已列出
